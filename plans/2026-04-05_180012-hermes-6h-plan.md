## CONTEXT FROM RESEARCH

### What the Claude Code Leak Revealed
- **1,902 files, 512,000+ lines across 29 subsystems** — $2.5B run-rate product
- 2 leaks in 1 week at Anthropic — "development velocity outrunning operational discipline"
- Theory: AI committed the build artifact (map file) that leaked its own code
- Key insight: **80% of Claude Code is plumbing** — session persistence, permissions, token budgets, checkpointing, event logging

### Hermes Specifics (from codebase research)
- Python pipeline scripts (decider-run, ai_decider, hl-sync-guardian, etc.) — **NO direct LLM calls**
- LLM calls happen in OpenClaw agent layer, not in Hermes trading pipeline
- `tool_use_enforcement: auto` in config.yaml applies to OpenClaw, not Hermes scripts
- Hermes scripts process data → database → API calls (Hyperliquid), not LLM tool calls
- `checkpoints:` section in config.yaml already exists (`enabled: true, max_snapshots: 50`) — for OpenClaw sessions, not trading pipeline

### What This Means for Hermes
The gap analysis in `upgrades.md` was correct but framed wrong for this system:
- Hermes doesn't have "tool loops" the same way Claude Code does — it runs scheduled pipeline scripts
- The equivalent risks are: runaway pipeline loops, duplicate orders on crash, session state loss
- The primitives we should port: **checkpoint hooks, workflow state, event logging, token budget enforcement**

## PHASE 1 — Tool Safety + Workflow State (Hour 1-2)

### 1.1 Add circuit breaker to decider-run.py
**File:** `/root/.hermes/scripts/decider-run.py`
**Risk:** Infinite loop, runaway position buildup, duplicate orders
**Approach:**
- Add `_tool_call_count` counter, reset per pipeline run
- Add `MAX_API_CALLS_PER_RUN = 30` constant
- Add `MAX_CONSECUTIVE_FAILURES = 5` (already exists as `MAX_CONSECUTIVE_FAILURES = 5` in hl-sync-guardian, but decider-run needs its own)
- Before each trade: check counter, abort if exceeded, log `CIRCUIT_BREAKER_TRIPPED`
- Before each trade: compute projected token budget (estimate from last N runs)
- On circuit breaker trip: emit structured stop reason, do NOT place order, log to event log

**Changes (exact lines to modify in decider-run.py):**
```python
# Add near top (line ~70)
MAX_API_CALLS_PER_RUN = 30
MAX_CONSECUTIVE_FAILURES = 3
_BUDGET_PER_RUN = 50000  # ~50k tokens per decider-run execution

_tool_call_count = 0
_consecutive_failures = 0
_session_token_budget_used = 0

# Add function after log()
def _check_circuit_breaker(action_name: str) -> bool:
    """Returns True if OK to proceed, False if circuit breaker tripped."""
    global _tool_call_count, _consecutive_failures
    if _tool_call_count >= MAX_API_CALLS_PER_RUN:
        log(f"[CIRCUIT_BREAKER] Max calls ({MAX_API_CALLS_PER_RUN}) reached, blocking {action_name}", 'WARN')
        return False
    if _consecutive_failures >= MAX_CONSECUTIVE_FAILURES:
        log(f"[CIRCUIT_BREAKER] Max failures ({MAX_CONSECUTIVE_FAILURES}) reached, blocking {action_name}", 'WARN')
        return False
    return True

def _record_call(success: bool, tokens_used: int = 0):
    global _tool_call_count, _consecutive_failures, _session_token_budget_used
    _tool_call_count += 1
    _session_token_budget_used += tokens_used
    if success:
        _consecutive_failures = 0
    else:
        _consecutive_failures += 1
```

Then wrap each trade placement in `_check_circuit_breaker()`.

### 2.3 Add recovery logic to decider-run startup
**File:** `/root/.hermes/scripts/decider-run.py`
**What:** On startup, check for incomplete run, reconstruct state if needed

```python
# At top of main():
def _check_incomplete_run():
    last = checkpoint_read_last('decider_run_start')
    if not last:
        return  # clean start
    
    # Was the last run incomplete?
    if last.get('workflow_state') not in ('IDLE', 'POSITION_OPEN'):
        # Previous run crashed mid-step
        log(f"[RECOVERY] Detected incomplete run from {last['ts']}", 'WARN')
        last_trade = checkpoint_read_last('trade_submitted')
        if last_trade:
            # Check if order actually filled — query HL
            hl_pos = get_open_hype_positions_curl()
            if last_trade['trade_id'] not in [p['trade_id'] for p in hl_pos]:
                log(f"[RECOVERY] Order {last_trade['trade_id']} not filled, clearing", 'WARN')
                # Mark trade as failed, don't duplicate
            else:
                log(f"[RECOVERY] Order {last_trade['trade_id']} IS filled, recovering position", 'WARN')
                # Set workflow_state to POSITION_OPEN, continue
        checkpoint_write('recovery_complete', {'from_step': last.get('step')})
```

event_log.py — Hermes event logging system

import json

def log_event(event_type: str, details: dict, level: str = 'INFO'):
    """Append structured event to event log."""
    entry = {
        "ts": datetime.utcnow().isoformat() + 'Z',
        "event": event_type,
        "level": level,

    # Rotation: if file > MAX_LOG_SIZE_MB, archive and start new
